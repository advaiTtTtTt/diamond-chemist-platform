## 🔴 Critical — Fix Before Delivery

### 1. PDF page count is broken for most real PDFs
**What they did:** Regex string match on raw buffer — `/\/Type\s*\/Page[^s]/g`

**Why it fails:** Any compressed, linearised, or encrypted PDF (which is most PDFs from phones, scanners, WhatsApp) will give wrong counts. A 10-page PDF might show as 3 pages. The chemist under-charges the customer. This is a billing error on every other order.

**Fix to give Antigravity:**
```
Replace the regex PDF page count with pdf-lib:

npm install pdf-lib  (or load from CDN in browser)

import { PDFDocument } from 'pdf-lib'

const countPdfPages = async (file) => {
  const arrayBuffer = await file.arrayBuffer()
  const pdf = await PDFDocument.load(arrayBuffer, { 
    ignoreEncryption: true 
  })
  return pdf.getPageCount()
}

This handles compressed, linearised, and most 
encrypted PDFs correctly. Use this exact code.
```

### 2. Agent race condition — double printing is real money lost
**What they admitted:** Two agent instances = same job printed twice = chemist pays for double paper + ink and customer gets charged once.

**Fix to give Antigravity:**
```
Add a database-level lock in agent-poll Edge Function.
Instead of SELECT then UPDATE separately, use 
a single atomic UPDATE...RETURNING:

const { data: jobs } = await supabase
  .from('print_jobs')
  .update({ 
    status: 'PRINTING',
    agent_version: agentVersion 
  })
  .eq('status', 'PAID')
  .order('paid_at', { ascending: true })
  .limit(5)
  .select()

This is an atomic operation. PostgreSQL row-level 
locking ensures only one agent claims each job.
The old SELECT → then UPDATE pattern is a race condition.
Remove agent-poll returning PAID jobs entirely.
The update itself IS the poll.
```

### 4. No file deletion — customer documents stored forever
**What they admitted:** Zero CRON job. Files stay in Storage indefinitely.

**Fix to give Antigravity:**
```
Create a new Edge Function: cleanup-print-files
Schedule it with pg_cron to run every 6 hours.

Logic:
  SELECT id, file_path FROM print_jobs
  WHERE status IN ('COLLECTED', 'CANCELLED', 'ERROR')
  AND created_at < NOW() - INTERVAL '48 hours'
  AND file_path IS NOT NULL

For each result:
  await supabase.storage
    .from('print-files')
    .remove([job.file_path])
  
  await supabase.from('print_jobs')
    .update({ file_path: null })
    .eq('id', job.id)

Enable pg_cron in Supabase dashboard:
  SELECT cron.schedule(
    'cleanup-print-files',
    '0 */6 * * *',
    'SELECT net.http_post(url, headers, body)'
  );
```

### 5. No tracking endpoint throttling
**What they admitted:** Anyone can brute-force all pickup codes.

**Fix to give Antigravity:**
```
In the tracking Edge Function, add IP-based rate limiting:

const clientIP = req.headers.get('x-forwarded-for') 
  || req.headers.get('cf-connecting-ip') 
  || 'unknown'

const { count } = await supabase
  .from('tracking_attempts')
  .select('*', { count: 'exact' })
  .eq('ip', clientIP)
  .gte('attempted_at', new Date(Date.now() - 600000).toISOString())

if (count >= 3) {
  return jsonResponse({ error: 'Too many attempts. Wait 10 minutes.' }, 429)
}

await supabase.from('tracking_attempts').insert({ ip: clientIP })

Create a simple tracking_attempts table:
  ip TEXT, attempted_at TIMESTAMPTZ DEFAULT now()
Add a pg_cron job to clean rows older than 1 hour.
```

# Diamond Chemist Print Agent — Setup Guide

## What this does
Runs silently on the shop computer. Automatically 
prints new orders as they arrive. No manual action needed.

## Requirements
- Windows 10/11 PC connected to the printer
- Node.js installed (download from nodejs.org)
- SumatraPDF installed (download from sumatrapdfreader.org)
  Add SumatraPDF to Windows PATH

## Setup (one time only)
1. Copy the agent/ folder to C:\DiamondChemistAgent\
2. Copy .env.example to .env
3. Open .env in Notepad and fill in:
   SUPABASE_URL → from Supabase dashboard
   AGENT_SECRET → from Supabase secrets
   PRINTER_NAME → exact name from Windows Settings 
                  → Bluetooth & devices → Printers
4. Open Command Prompt in the folder
5. Run: npm install
6. Run: node print-agent.js
   You should see: "Diamond Chemist Print Agent running..."

## How to find your exact printer name
Windows key → Settings → Bluetooth & devices → Printers & scanners
Copy the exact name shown (e.g. "HP LaserJet Pro M404dn")
Paste into PRINTER_NAME in .env

## Making it start automatically with Windows
Run install-service.bat as Administrator.
The agent will start silently on every reboot.

## If something goes wrong
Check C:\DiamondChemistAgent\agent.log
The error message will tell you what failed.
